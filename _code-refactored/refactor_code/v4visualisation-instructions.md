# V4 Visualisation — Repeatable Render Instructions

## Renderer

`_code-refactored/refactor_code/outputs/report/render_proposal_v4.py`

Renders hybrid proposal views from `state_with_indicators.vtk` files. The hybrid view colours the decay pathway (senescing, snag, fallen, decayed) by forest_size colour and only applies proposal overlays on non-decay voxels. Output is a PNG with an integrated bottom legend.

## Prerequisites

- VTKs must already exist (generated by the batch runner or equivalent)
- The renderer does **not** generate VTKs — it only reads them
- Uses `uv run python` (repo-local `.venv`)

## Key Environment Variable

```
REFACTOR_RUN_OUTPUT_ROOT=<path-to-run-root>
```

This controls where VTKs are read from and where renders are written. The renderer resolves paths via `refactor_code.paths`:

- **Scenario VTKs**: `{root}/output/vtks/{site}/{site}_{scenario}_{voxel}_yr{year}_state_with_indicators.vtk`
- **Baseline VTKs**: `{root}/output/vtks/{site}/{site}_baseline_{voxel}_state_with_indicators.vtk`
- **Render output**: `{root}/temp/validation/renders/custom/`

## Shared Layout Parameters

These ensure consistent model size and vertical alignment across sites:

- `--model-base-y 1170` — aligns the bottom of all models to the same vertical position
- `--target-model-width 1757` — normalises apparent model size across sites with different camera distances

These were pre-computed from measuring all sites. Pass them to skip the measurement pass and enable parallel execution.

## Render Commands

### Single site + scenario + year

```bash
REFACTOR_RUN_OUTPUT_ROOT=<root> uv run python \
  _code-refactored/refactor_code/outputs/report/render_proposal_v4.py \
  --site trimmed-parade --scenario positive --years 180 \
  --output-mode validation --hybrid-only \
  --model-base-y 1170 --target-model-width 1757
```

### All scenarios for one site (includes baseline)

```bash
REFACTOR_RUN_OUTPUT_ROOT=<root> uv run python \
  _code-refactored/refactor_code/outputs/report/render_proposal_v4.py \
  --site trimmed-parade --scenario all \
  --years 0 10 30 60 90 120 150 180 \
  --output-mode validation --hybrid-only \
  --model-base-y 1170 --target-model-width 1757
```

### All sites in parallel

Run 3 commands concurrently, one per site:

```bash
for site in trimmed-parade city uni; do
  REFACTOR_RUN_OUTPUT_ROOT=<root> uv run python \
    _code-refactored/refactor_code/outputs/report/render_proposal_v4.py \
    --site $site --scenario all \
    --years 0 10 30 60 90 120 150 180 \
    --output-mode validation --hybrid-only \
    --model-base-y 1170 --target-model-width 1757 &
done
wait
```

## Sites

- `trimmed-parade`
- `city`
- `uni`

Camera presets per site are in `render_forest_size_views.py` → `CAMERAS` dict.

## Scenarios

- `positive`
- `trending`
- `baseline` (no year — resolves to yr0 in output filename)

## Years

`0, 10, 30, 60, 90, 120, 150, 180`

## Full render set

3 sites x (2 scenarios x 8 years + 1 baseline) = **51 images**

## Output filenames

`{site}_{scenario}_yr{year}_engine3-proposal-hybrid_with-legend.png`

## Flags reference

| Flag | Purpose |
|------|---------|
| `--site` | Site name or `all` |
| `--scenario` | `positive`, `trending`, `baseline`, or `all` |
| `--years` | Space-separated year list |
| `--output-mode` | `validation` (use this) |
| `--hybrid-only` | Skip non-hybrid render variants |
| `--model-base-y` | Shared vertical alignment (use `1170`) |
| `--target-model-width` | Shared width normalisation (use `1757`) |

## Polling for fresh VTKs

If VTKs are being regenerated by a batch run, check timestamps before rendering. A site needs 16 scenario VTKs (8 years x 2 scenarios) all newer than the batch start time. Baselines are separate (`--baselines-only` from the batch runner).
