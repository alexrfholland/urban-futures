# engine_v3.py change log — 2026-04-10 16:11

**This commit touches 1 file (`engine_v3.py`), but it actually contains three
logically independent changes bundled together. The one to look out for is
the recruit refactor — everything else is incidental.**

Line numbers below refer to the working-tree state *after* this commit.

---

## 1. Recruit refactor (the thing to look out for)

Rewrites the recruitment system from per-parent-canopy proximity placement to
a zone-mask + voxel-quota model with three explicit mechanisms.

### New
- **`RECRUIT_MECHANISMS` constant** — lines 88–112
  Config table for the three mechanisms (`node-rewild`, `under-canopy`,
  `ground-rewild`). Each entry declares its `zone_field`,
  `ID_for_linking_df_to_xarray`, `under_node_treatment`,
  `recruit_intervention`, and `recruit_mechanism` label.

- **`_collect_active_parent_ids`** — new helper
  Returns the integer `NodeID`s of non-absent trees with a given
  `under-node-treatment`. Used to build zone masks.

- **`_compute_recruit_zone_masks`** — new helper
  Single source of truth for all three zone masks. Given the tree df, the
  dataset, and the depaved threshold, returns boolean voxel masks for
  `node-rewild`, `under-canopy`, `ground-rewild`, plus `rewilding-enabled`
  (all depaved ground, pre-partition). Reused by `log_run_stats` so runtime
  and post-hoc area reporting stay consistent.

- **`_recruit_occupancy_by_type`** — replaces `_recruit_occupancy_counts`
  Counts alive recruits grouped by `recruit_mechanism` (`node-rewild`,
  `under-canopy`, `ground`) instead of keying by `recruit_source_id`.

### Rewritten
- **`_recruit_active_mask`** — line 1194
  Now excludes `senescing` from the "occupies space for spacing checks" set.
  Previously: `{small, medium, large, senescing}`. Now: `{small, medium, large}`.

- **`_append_recruit_telemetry_csv`** — line 1207
  New fieldnames match the new per-mechanism-per-pulse telemetry shape:
  `pulse_duration`, `density_per_pulse`, `type`, `total_area`,
  `zone_voxel_count`, `quota`, `occupancy`, `to_place`, `placed`, `filled`,
  `unfilled`, `rejected_spacing`, `attempts`. Drops `source_id`,
  `source_treatment`, `candidate_count`, `starting_occupancy`,
  `remaining_capacity`, `rejected_attempt_limit`. Writer uses
  `extrasaction="ignore"` so upstream callers can stuff extra fields in.

- **`calculate_under_node_treatment_status`** — line 1418
  Was: KDTree proximity filter against living trees, producing a single
  `scenario_rewildingPlantings` mask by excluding voxels within
  `RECRUIT_SPACING_THRESHOLD_METERS` of any existing tree.
  Now: delegates to `_compute_recruit_zone_masks` and persists four masks
  onto `ds`:
    - `scenario_rewildingEnabled` (all depaved ground)
    - `scenario_nodeRewildRecruitZone`
    - `scenario_underCanopyRecruitZone`
    - `scenario_rewildGroundRecruitZone`
  The old `scenario_rewildingPlantings` field is gone. Year-0 stamps all
  four masks to `-1`.

- **`apply_recruit`** — line 1467
  Complete rewrite. Key behavioral changes:
    - Local `rng = np.random.default_rng(seed + 3000 + year)` replaces
      global `np.random.seed(...)`.
    - Unified loop over `RECRUIT_MECHANISMS` instead of separate node vs
      ground code paths.
    - Quota per mechanism is derived from the zone mask:
      `round(zone_voxel_count * voxel_area * density_per_pulse)` — i.e. a
      single zone-wide budget per mechanism. Previously node-based
      mechanisms quota'd per-parent via `sim_NodesArea` / `CanopyArea`.
    - Primary placement: shuffle zone voxels, walk the list, reject on
      spacing against all accepted positions.
    - Fallback (node-based mechanisms only): weighted random draw over
      parents (weights = parent's owned voxel count in the zone), offset
      position, snap to nearest zone voxel.
    - Spacing threshold is fixed at `RECRUIT_SPACING_THRESHOLD_METERS`
      throughout (no more relaxed-threshold retry pass).
    - Telemetry now emits exactly one row per mechanism per pulse, keyed by
      `type = mechanism_key`.

### Unchanged (worth noting)
- **`_make_new_tree_record`** body — same signature, same output schema.
- **`apply_annual_tree_mortality`** mortality *behavior* — unchanged
  (but see change #3 below for added debug columns).
- **`deploy_ground_deadwood`** does not go through `apply_recruit`; it's a
  separate feature (change #2).
- **Seed offsets** for other RNG users (growth=+6000, mortality=+2000,
  decay=+0, decay_lifecycle=+1000, replant=+5000, stagger=+4000) —
  untouched. Only recruit moved from global-seed to local-rng.

## 2. Ground-deadwood deploy proposal (bundled, not part of recruit)

New end-of-pulse pass that places fallen/decayed deadwood features onto the
ground-rewild zone. **Only runs when a scenario has `deployGroundDeadwood`
in params with `enabled: True`** — currently only `uni_positive`.

- **`_DEADWOOD_VOLUME_LOOKUP` + `_load_deadwood_volume_lookup`** — line 1757
  Cached loader for the pre-built template volume lookup CSV under
  `_data-refactored/model-inputs/tree_variants/.../trees/template-edits_base_geometry_volume_lookup.csv`.

- **`deploy_ground_deadwood`** — line 1776
  Reads `scenario_rewildGroundRecruitZone` (the same zone recruit uses),
  computes a volume budget from `deadwood_m3_per_ha * fallen_share *
  eligible_area_ha * max_fraction_per_pulse`, picks random templates until
  the budget is hit, then places them on shuffled zone voxels. Soft spacing
  against existing trees (`spacing_preference` metres); if a candidate
  fails spacing it's pushed to a fallback queue rather than rejected.
  Fallen records get `became_fallen_at_year` + `fallen_decay_after_years`;
  decayed records get `became_decayed_at_year` +
  `decayed_remove_after_years`. Uses RNG seed offset `+7000`.

- **`_run_single_pulse`** — line 1979
  Adds `df = deploy_ground_deadwood(df, pulse_ds, params)` immediately
  after `apply_recruit`. No-op for any scenario without the params block.

**Note:** the corresponding `DEPLOY_GROUND_DEADWOOD` constant in
`constants.py` and the `deployGroundDeadwood` params block in
`params_v3.py::params_uni_positive` are **not** included in this commit —
they remain as working-tree modifications. The deadwood feature will not
run until those are also committed.

## 3. Run stats / telemetry reporter + mortality debug columns

Offline reporting helper, plus per-cohort mortality debug columns that
were added while investigating the v4.7→v4.75 divergence. Neither affects
simulation behavior; both are pure bookkeeping.

### Stats reporter
- **Import** — line 30
  Adds `scenario_output_root` to the imports from `refactor_code.paths`.

- **`DEFAULT_BASELINE_ROOTS`** — line 2093
  Dict mapping site → default baseline root path, used when a run's own
  root has no baseline nodeDF.

- **`_find_baseline`** — line 2102
  Walks current root, then default root, then legacy fallback names
  (`v4-3`, `v4-allometric-flat-mortality`, `v4updatedterms`) looking for
  `{site}_baseline_1_nodeDF_-180.csv`.

- **`log_run_stats`** — line 2121
  Produces `{site}_{scenario}_recruit_stats.csv`,
  `{site}_{scenario}_size_stats.csv`, and appends a per-site/scenario
  row-set into `{root}/comparison/interim-size-assessment.csv`. Also
  prints formatted tables to stdout. Consumes the recruit telemetry CSV
  if present, otherwise falls back to counting placed recruits from the
  tree DF. Uses `_compute_recruit_zone_masks` (change #1) as the source
  of truth for zone areas.

### Mortality debug columns
- **`apply_annual_tree_mortality`** — lines 637–642, 663–664
  Adds two columns and populates them inside the cohort loop:
    - `recruit_mortality_rate` — the `step_death_probability` applied to
      each tree's cohort
    - `recruit_mortality_cohort` — the integer cohort index
      (`floor(dbh/10)`) that the tree belonged to
  **The mortality math itself is unchanged** — same
  `rng.choice(cohort_index, size=death_count)`, same cohort ordering,
  same annual-rate halving above cohort 8. The columns are purely
  informational and are overwritten each year. The `recruit_` prefix is
  a misnomer inherited from the debugging session; they apply to every
  tree, not just recruits.
